# 需求说明

## 1. 产品定位

Movie Tool 是一个局域网媒体管理服务，用于管理 NAS、本地磁盘、远程挂载目录、网盘 STRM 目录中的媒体文件。系统应覆盖电影、电视剧、动漫、AV 四大类型，并提供刮削、缓存、搜索、合集、人物标签、多版本、NFO、STRM、Emby/Jellyfin/Plex 兼容能力。

## 2. 用户场景

- 用户在 NAS 上部署服务，通过浏览器、桌面端或移动端访问。
- 用户配置一个或多个媒体库路径。
- 系统首次全量扫描已有文件，之后监听目录增量变化。
- 新增文件后，系统解析文件名、识别媒体类型、刮削元数据。
- 当刮削结果不确定时，前端提示用户手动选择。
- 元数据写入数据库缓存，可选择写回媒体目录。
- 系统生成 NFO、图片、STRM，并刷新 Emby/Jellyfin/Plex。
- 用户可按标题、演员、女优、导演、片商、标签、合集、版本、清晰度查询媒体。

## 3. 媒体类型

### 3.1 普通媒体

- 电影
- 电视剧
- 动漫
- 纪录片
- 综艺，可作为后续扩展

### 3.2 AV 媒体

- 有码
- 无码
- FC2
- 无码流出
- VR，可作为后续扩展
- 多分片、多版本、多字幕版本

## 4. 功能需求

### 4.1 媒体库

- 支持多个媒体库。
- 每个媒体库可配置类型、路径、语言、刮削源优先级、缓存策略、NFO 策略、STRM 策略。
- 支持路径映射，用于 Docker 容器路径与宿主机/媒体服务器路径转换。

### 4.2 文件扫描与监听

- 支持全量扫描。
- 支持增量监听。
- 支持文件新增、修改、删除、移动。
- 文件消失后先标记为 `missing`，不立即删除数据库记录。
- 支持自动清理策略，例如 missing 超过 N 天后清理。

### 4.3 文件整理

- 支持将相同媒体归入同一个作品文件夹。
- 支持电影、电视剧、动漫、AV 分别配置整理命名规则。
- 整理前必须生成预览计划，展示源路径、目标路径、冲突、风险。
- 默认 dry-run，不直接移动文件。
- 支持移动、复制、硬链接、软链接四种策略。
- 支持连带整理字幕、NFO、图片、预告片、额外文件。
- 支持同一媒体的多个版本进入同一个文件夹。
- 支持整理后更新数据库中的文件路径。
- 支持跨库整理，例如下载目录整理到媒体库目录。
- 支持失败回滚记录，至少能追踪每次移动动作。
- 支持冲突处理策略：跳过、重命名、覆盖需二次确认。

推荐目录结构：

```text
Movies/Inception (2010)/
  Inception (2010) - 2160p Remux.mkv
  Inception (2010) - 1080p BluRay.mkv
  Inception (2010).zh-CN.srt
  poster.jpg
  fanart.jpg
  movie.nfo

TV/Breaking Bad (2008)/
  Season 01/
    Breaking Bad - S01E01 - 1080p WEB-DL.mkv

AV/ABC-123 Title/
  ABC-123 - 1080p.mp4
  ABC-123 - 2160p.mp4
  poster.jpg
  movie.nfo
```

### 4.4 刮削与正确性检测

- 刮削结果必须经过匹配评分。
- 高置信度可自动匹配。
- 多候选、低置信度、版本冲突时进入人工确认。
- 用户确认后的匹配应锁定，避免后续自动覆盖。
- 保存候选结果与决策记录，方便追溯。

### 4.5 国际化与翻译

- 默认语言为 `zh-CN`。
- 支持媒体库级、媒体级语言覆盖。
- 中文缺失时按语言优先级 fallback。
- AV 媒体中文缺失时，必须对标题、简介、标签、系列名等进行翻译。
- 默认展示中文译文，同时保留原文。
- 用户手动编辑的译文应锁定。

### 4.6 AI 功能

- 支持配置 AI Provider、模型、API Key、Base URL。
- 支持 OpenAI-compatible API。
- AI 可用于候选判断、文件名解析、翻译、简介生成、标签规范化、番号纠错。
- AI 结果必须可解释，保存推荐原因与置信度。
- AI 功能默认可关闭。

### 4.7 多版本管理

- 一个作品可包含多个版本。
- 版本字段包括清晰度、片源、编码、HDR、音轨、字幕、发布组、文件大小、码率、容器、是否默认。
- 支持本地文件版本与 STRM 版本同时存在。
- 前端支持设置默认版本、查看缺失版本、重新识别版本。

### 4.8 合集、人物、标签

- 支持官方合集、自动合集、手动合集、智能规则合集。
- 普通媒体支持按演员、导演、编剧、配音、公司、类型、标签查询。
- AV 支持按女优、男优、导演、片商、发行商、系列、番号前缀、标签查询。
- 支持人物别名、标签同义词、片商别名。

### 4.9 Emby/Jellyfin/Plex 兼容

- 支持生成 NFO、poster、fanart、thumb 等文件。
- 支持刷新 Emby/Jellyfin/Plex 媒体库。
- 支持按不同服务器适配人物、标签、片商、合集字段映射。

### 4.10 STRM

- 支持从本地路径或远程路径生成 STRM。
- 支持 WebDAV、Alist、网盘映射等后续扩展。
- 支持路径映射、批量生成、增量更新、失效检测。

### 4.11 自动化管理

- 支持创建、启用、暂停、删除自动化规则。
- 支持按固定间隔或 cron 表达式触发任务。
- 支持自动化任务类型：扫描媒体库、刮削待处理媒体、翻译缺失中文、生成 NFO、生成 STRM、整理文件、刷新媒体服务器、清理 missing 文件。
- 支持自动化作用范围：全部媒体库、指定媒体库、指定媒体类型、指定筛选条件。
- 支持并发限制、失败重试、静默时间窗口。
- 支持“只生成计划不执行”的安全模式，特别用于文件整理。
- 支持运行历史、最近错误、下次运行时间。
- 自动化触发后应生成普通任务记录，便于在任务中心统一查看。

## 5. 非功能需求

- 后端应支持 Docker 部署。
- API 应可被 Flutter Web/Desktop/Mobile 共用。
- 所有长耗时操作应进入任务系统。
- 支持代理、限速、重试。
- 关键操作应保留日志。
- 数据库可备份与迁移。
