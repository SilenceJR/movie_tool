# 前端设计

## 1. 技术方向

- Flutter Web / Desktop / Mobile 共用业务层。
- 状态管理建议 Riverpod。
- API 客户端建议 Dio。
- UI 使用 Material 3。

## 2. 页面结构

```text
仪表盘
媒体库
媒体浏览
媒体详情
待确认
任务中心
自动化
合集
人物
标签
片商/组织
STRM
文件整理
外部服务器
AI 与翻译
系统设置
```

## 3. 关键页面

### 3.1 仪表盘

展示：

```text
媒体总数
今日新增
待确认数量
失败任务
missing 文件
最近刮削
最近任务
```

### 3.2 待确认

用于处理 ambiguous / low_confidence。

布局：

```text
左侧：本地文件与解析信息
右侧：候选卡片列表
底部：手动搜索、填写外部 ID、忽略、锁定
```

候选卡片：

```text
海报
标题
年份
来源
匹配分
匹配原因
冲突提示
AI 推荐标记
```

### 3.3 媒体详情

包含：

```text
基础元数据
图片
版本列表
文件列表
人物
标签
组织
合集
NFO 状态
STRM 状态
服务器同步状态
```

### 3.4 人物页

普通媒体：

```text
演员、导演、编剧、配音作品
```

AV：

```text
女优、男优、导演作品
合作片商
常见系列
标签分布
```

### 3.5 高级搜索

筛选条件：

```text
媒体类型
人物
标签
合集
组织/片商
年份
清晰度
片源
HDR
字幕
文件状态
匹配状态
```

### 3.6 文件整理

用于把相同媒体归入同一个作品文件夹。

页面能力：

```text
整理规则列表
目标目录配置
电影/剧集/动漫/AV 命名模板
生成整理预览
源路径与目标路径对比
冲突提示
sidecar 文件识别
执行进度
历史动作
```

整理预览必须清楚展示：

```text
将移动/复制/链接多少文件
会创建多少目录
有多少冲突
有多少文件会被跳过
执行后是否更新数据库路径
```

### 3.7 自动化

页面能力：

```text
自动化规则列表
创建/暂停/恢复/删除
立即运行
下次运行时间
最近运行状态
失败原因
运行历史
```

可配置任务：

```text
扫描媒体库
刮削待处理媒体
翻译缺失中文
文件整理
生成 NFO
生成 STRM
刷新 Emby/Jellyfin/Plex
清理 missing 文件
```

## 4. 交互规则

- 所有危险操作需要二次确认。
- 刮削候选选择后可以锁定。
- 用户编辑字段默认标记为 manual_edited。
- 任务运行中应实时显示进度。
- 大批量操作应显示预计影响数量。
